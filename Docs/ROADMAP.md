# 开发路线图

## 目标

开发一个 iOS App，用于浏览 `https://e-hentai.org/` 上与搜索、详情和阅读有关的主要功能。

## 阶段拆分

### 阶段 1：工程初始化

- 初始化 Git 仓库。
- 建立 SwiftUI iOS 工程骨架。
- 建立中文界面占位。
- 建立基础文档。
- 验证工程可构建。

### 阶段 2：站点结构调研与网络层

- 查证当前站点搜索页、详情页、阅读页结构。
- 定义搜索结果、图库详情、图片页等领域模型。
- 实现基础 HTTP 客户端、Cookie 持久化和错误模型。
- 编写解析层测试样例。

当前进度：已完成公开页面结构记录、领域模型、HTTP 客户端、Cookie 注入和解析层测试，网络层已服务搜索、详情和阅读闭环。

### 阶段 3：搜索功能

- 支持关键词搜索。
- 支持分页。
- 支持分类、评分、页数、上传时间等主要信息展示。
- 支持高级搜索参数。

当前进度：已支持首页/热门/关注/收藏浏览来源、关键词搜索、隐藏分类、高级搜索参数、筛选重置、筛选展开后的整页滚动、结果列表、缩略图、上传者、发布时间、标签摘要、上一页/下一页、按站点 `jump` 参数页码跳转、最近搜索词复用和保留筛选参数的失败重试。分页按钮使用图标按钮以适配窄屏。关注和收藏来源需要用户在设置页配置站点 Cookie。

### 阶段 4：图库详情

- 展示标题、封面、标签、上传信息、评分、页数等信息。
- 支持从详情页进入阅读器。
- 支持缩略图分页。

当前进度：已支持从搜索结果进入详情页，展示核心元信息、标签和带缩略图的阅读入口，标签默认折叠且可展开后继续发起搜索，并可按页或一次性加载后续缩略图分页来补全更多阅读入口；进入图库时会显示轻量动画加载提示；信息区域默认折叠，封面和页面预览以静态图展示 GIF，已缓存页面会优先用本地缓存原图作为预览；详情页可区分本地收藏与线上收藏，有 Cookie 时可通过站点收藏弹窗加入或取消线上收藏，并在已加入线上收藏时于顶部显示状态；详情页提供整本下载按钮，下载会在后台缓存全部已解析阅读页并显示已下载/总页数进度，进度读取会按实际图库缓存记录校准；详情加载失败时提供重试入口，如果该图库已有缓存页则会显示本地缓存阅读入口。

### 阶段 5：阅读器

- 支持单页阅读。
- 支持上一页、下一页和跳页。
- 支持图片加载状态、错误状态和重试。
- 支持基础阅读设置。

当前进度：已支持从详情页进入全屏阅读器、返回上一级、默认沉浸式单图阅读、左/右点击翻页、中间点击显示控件、长按确认后保存当前图片到系统相册、加载单张图片、上一页、下一页、已知页面缩略图目录、图片页请求失败重试、图片资源加载失败重试、GIF 动图渲染、已查看图片缓存、翻页时优先用缓存索引恢复已缓存阅读页且不重新请求 HTML、离线时通过缓存索引恢复已缓存阅读页、未缓存/失败页仍可通过已知目录继续上一页或下一页、双指缩放、手动横竖屏切换、当前页/图库页/原图链接入口、上次阅读恢复、页码输入跳转、跳页前自动补齐图库缩略图目录、已知页码范围提示、图片显示模式、缩放倍率和阅读背景偏好。

### 阶段 6：体验补齐

- 历史记录。
- 本地收藏。
- 阅读进度。
- 设置页。
- 站点 Cookie 配置。
- 文档补齐和回归验证。

当前进度：已实现本地书架 Tab、历史记录、本地收藏、线上收藏入口、最近阅读页持久化、从书架直接继续阅读，以及设置页里的站点 Cookie、应用主题模式、自定义主题颜色、阅读偏好、本地数据统计、图片缓存统计、按图库缓存管理、单图库缓存删除、非图库图片缓存清理和全部清空入口。图片缓存区域位于本地数据区域之前；缓存管理页支持列表/网格切换、标题或备注筛选，并可将所有缓存图库按图库文件夹永久迁移到 Files 可见的 `Cached Gallery` 目录；迁移使用独立暂存目录，验证完成后才替换正式目录和删除重复缓存。后续明确触发的整本下载直接写入永久暂存区，普通浏览图片继续使用可清理缓存。缓存管理页可一键继续所有未完成图库下载，最多同时运行 5 个图库任务，并在有任务时显示总体下载页数进度、下载网速、活动任务数和排队任务数，任务进行中可暂停所有下载；点击本地图库会先进入缓存入口页，可不依赖远端详情页直接打开阅读器；单页下载任务按完成顺序补位，失败会随机短暂延迟后重试，仍失败时跳过该页继续后续页面；遇到 HTTP 404 的图库会在本地索引中标记，后续批量继续未完成下载时会自动跳过，但不影响继续阅读已有本地页。图片预览会按目标尺寸降采样，减少本地原图作为缩略图时的内存占用和滚动卡顿。线上收藏只保留关键词搜索，并默认请求全部收藏分类；书架顶部区域不再固定，线上收藏上一页/下一页放在列表底部以保持和首页搜索结构一致。详情页和书架的阅读入口会打开全屏阅读器并保留原底部 Tab 状态；阅读器可在已加载或自动补齐的页面目录中按页码跳转，并可保存阅读器缩放偏好；图库详情、缓存管理等二级页面会隐藏底部 Tab。

### 阶段 7：验证与打磨

- 自动化测试。
- 模拟器启动冒烟。
- 视觉与交互问题修正。

当前进度：已完成自动化测试、Debug 构建、模拟器安装启动和首屏截图复测，记录见 `Docs/QA.md`。
