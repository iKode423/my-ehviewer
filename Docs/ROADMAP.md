# 开发路线图

## 目标

开发一个 iOS App，用于浏览 `https://e-hentai.org/` 上与搜索、详情和阅读有关的主要功能。

## 阶段拆分

### 阶段 1：工程初始化

- 初始化 Git 仓库。
- 建立 SwiftUI iOS 工程骨架。
- 建立中文界面占位。
- 建立基础文档。
- 验证工程可构建。

### 阶段 2：站点结构调研与网络层

- 查证当前站点搜索页、详情页、阅读页结构。
- 定义搜索结果、图库详情、图片页等领域模型。
- 实现基础 HTTP 客户端、Cookie 持久化和错误模型。
- 编写解析层测试样例。

当前进度：已完成公开页面结构记录、领域模型、HTTP 客户端、Cookie 注入和解析层测试，网络层已服务搜索、详情和阅读闭环。

### 阶段 3：搜索功能

- 支持关键词搜索。
- 支持分页。
- 支持分类、评分、页数、上传时间等主要信息展示。
- 支持高级搜索参数。

当前进度：已支持首页/热门/关注/收藏浏览来源、关键词搜索、隐藏分类、高级搜索参数、筛选重置、筛选展开后的整页滚动、结果列表、缩略图、上传者、发布时间、标签摘要、上下页、最近搜索词复用和保留筛选参数的失败重试。关注和收藏来源需要用户在设置页配置站点 Cookie。

### 阶段 4：图库详情

- 展示标题、封面、标签、上传信息、评分、页数等信息。
- 支持从详情页进入阅读器。
- 支持缩略图分页。

当前进度：已支持从搜索结果进入详情页，展示核心元信息、标签、站点网页入口和带缩略图的阅读入口，标签可继续发起搜索，并可按页或一次性加载后续缩略图分页来补全更多阅读入口；信息区域默认折叠，封面和页面预览以静态图展示 GIF；详情页可区分本地收藏与网站收藏，有 Cookie 时可通过站点收藏弹窗同步线上收藏；详情加载失败时提供重试入口。

### 阶段 5：阅读器

- 支持单页阅读。
- 支持上一页、下一页和跳页。
- 支持图片加载状态、错误状态和重试。
- 支持基础阅读设置。

当前进度：已支持从详情页进入阅读器、默认沉浸式单图阅读、左/右点击翻页、中间点击显示控件、加载单张图片、上一页、下一页、已知页面缩略图目录、图片页请求失败重试、图片资源加载失败重试、GIF 动图渲染、已查看图片缓存、双指缩放、手动横竖屏切换、当前页/图库页/原图链接入口、上次阅读恢复、页码输入跳转、跳页前自动补齐图库缩略图目录、已知页码范围提示、图片显示模式、缩放倍率和阅读背景偏好。

### 阶段 6：体验补齐

- 历史记录。
- 本地收藏。
- 阅读进度。
- 设置页。
- 站点 Cookie 配置。
- 文档补齐和回归验证。

当前进度：已实现本地书架 Tab、历史记录、本地收藏、网站收藏入口、最近阅读页持久化、从书架直接继续阅读，以及设置页里的站点 Cookie、应用主题模式、阅读偏好、本地数据统计、图片缓存统计和清空入口。详情页和书架的阅读入口会同步切换到底部“阅读”Tab 并带入当前阅读会话；阅读器可在已加载或自动补齐的页面目录中按页码跳转，并可保存阅读器缩放偏好。

### 阶段 7：验证与打磨

- 自动化测试。
- 模拟器启动冒烟。
- 视觉与交互问题修正。

当前进度：已完成自动化测试、Debug 构建、模拟器安装启动和首屏截图复测，记录见 `Docs/QA.md`。
