# 开发路线图

## 目标

开发一个 iOS App，用于浏览 `https://e-hentai.org/` 上与搜索、详情和阅读有关的主要功能。

## 阶段拆分

### 阶段 1：工程初始化

- 初始化 Git 仓库。
- 建立 SwiftUI iOS 工程骨架。
- 建立中文界面占位。
- 建立基础文档。
- 验证工程可构建。

### 阶段 2：站点结构调研与网络层

- 查证当前站点搜索页、详情页、阅读页结构。
- 定义搜索结果、图库详情、图片页等领域模型。
- 实现基础 HTTP 客户端、Cookie 持久化和错误模型。
- 编写解析层测试样例。

当前进度：已完成公开页面结构记录、领域模型、HTTP 客户端和解析层测试。下一步接入搜索界面闭环。

### 阶段 3：搜索功能

- 支持关键词搜索。
- 支持分页。
- 支持分类、评分、页数、上传时间等主要信息展示。
- 支持高级搜索参数。

当前进度：已支持首页/热门/关注/收藏浏览来源、关键词搜索、隐藏分类、高级搜索参数、结果列表、缩略图、上传者、发布时间、标签摘要、上下页、最近搜索词复用和保留筛选参数的失败重试。关注和收藏来源需要用户在设置页配置站点 Cookie。

### 阶段 4：图库详情

- 展示标题、封面、标签、上传信息、评分、页数等信息。
- 支持从详情页进入阅读器。
- 支持缩略图分页。

当前进度：已支持从搜索结果进入详情页，展示核心元信息、标签和当前缩略图页中的阅读入口，标签可继续发起搜索，并可按页或一次性加载后续缩略图分页来补全更多阅读入口；详情加载失败时提供重试入口。

### 阶段 5：阅读器

- 支持单页阅读。
- 支持上一页、下一页和跳页。
- 支持图片加载状态、错误状态和重试。
- 支持基础阅读设置。

当前进度：已支持从详情页进入阅读器、加载单张图片、上一页、下一页、错误重试、原图入口、上次阅读恢复、已知页面入口页码输入跳转、已知页码范围提示、图片显示模式、缩放倍率和阅读背景偏好。

### 阶段 6：体验补齐

- 历史记录。
- 本地收藏。
- 阅读进度。
- 设置页。
- 站点 Cookie 配置。
- 文档补齐和回归验证。

当前进度：已实现本地书架 Tab、历史记录、收藏切换、最近阅读页持久化，以及设置页里的站点 Cookie、阅读偏好、本地数据统计和清空入口。详情页补全全部页面入口后，阅读器可以在全部已加载页面之间按页码跳转，并可保存阅读器缩放偏好。

### 阶段 7：验证与打磨

- 自动化测试。
- 模拟器启动冒烟。
- 视觉与交互问题修正。

当前进度：已完成一次模拟器安装、启动和首屏截图检查，记录见 `Docs/QA.md`。
