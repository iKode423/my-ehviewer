# 架构说明

## 分层

MyEHViewer 会按以下边界逐步建设：

- `App`：应用入口、根导航、全局主题偏好和通用文案。
- `Features/Search`：搜索输入、结果列表、高级搜索入口。
- `Features/Gallery`：图库详情、标签和元信息展示。
- `Features/Reader`：阅读器、图片页导航和阅读状态。
- `Networking`：HTTP 请求、Cookie header、请求头、响应错误和图片缓存。
- `Parsing`：HTML 到领域模型的解析。
- `Persistence`：历史、收藏和设置。

## 当前状态

当前仓库包含工程骨架、领域模型、HTTP 客户端、基础解析层、搜索界面闭环、图库详情页、基础阅读器和本地书架。解析层已覆盖搜索列表、图库详情和阅读图片页的核心结构，并使用中性 HTML fixture 测试。

根导航由 `ContentView` 持有 `AppNavigationStore`，统一管理底部 Tab selection 和当前 `ReaderRoute`。阅读器不再是底部 Tab，而是从详情页、书架继续阅读和缓存管理等入口以全屏子页面呈现；关闭阅读器后会回到原来的上一级页面，避免底部 Tab 选中状态和当前阅读会话错位。

搜索界面由 `SearchViewModel` 负责浏览来源、筛选状态、请求、解析、分页、初始关键词搜索、初始浏览来源、最近搜索、失败重试 URL 和错误状态，`SearchView` 只负责 SwiftUI 展示与用户输入，并可在根页面和详情页导航栈复用。书架线上收藏复用同一个 `SearchViewModel(initialSource: .favorites)`，但把关键词搜索和上一页/下一页固定在书架分段控件下方，列表滚动时系统大标题会收起以腾出空间。首页、热门、关注和站点收藏来源都通过 `EHSearchRequest` 生成 URL，再复用同一解析流程。搜索结果行会撑满列表宽度，避免不同标题长度导致缩略图和文字横向错位。

图库详情页由 `GalleryDetailViewModel` 负责详情请求、解析、缩略图分页合并和线上收藏表单提交，`GalleryDetailView` 展示封面、默认折叠的元信息、可继续搜索的标签、带缩略图的阅读页入口、后台下载入口和失败重试操作。收藏工具栏用菜单区分本地收藏和线上收藏；线上收藏会先请求 `gallerypopups.php` 弹窗，解析表单 action、隐藏字段、已选分类和备注字段，再用 URL-encoded POST 提交，取消收藏时提交站点使用的移除分类值，Cookie 仍由共享 HTTP 客户端注入。图库页面预览会优先使用对应页的本地缓存原图；未缓存时才使用站点缩略图和 CSS sprite 裁剪数据。

阅读器由 `ReaderViewModel` 负责图片页请求、解析、翻页、已知页码范围、页面目录自动补齐、页码入口跳转状态和图片资源重载 token，`ReaderView` 展示当前图片、返回上一级、页码、上一页、下一页、缩略图目录、页码输入跳转、图片加载失败重试、缩放控制和显示偏好，顶部工具栏不再暴露外部浏览器入口。阅读器默认隐藏主要控制区，只显示图片和一个左上角返回目标；用户点击左侧/右侧翻页，点击中间显示控制 UI，并可通过双指缩放临时放大。上一页/下一页会按当前页码优先查已知图库目录和缓存索引，再回退到阅读页 HTML 解析出的相邻链接，因此只有第一页/最后一页才应禁用相应按钮。远端图片通过 `CachedRemoteImageView` 统一加载，先读 `ImageCacheStore` 的磁盘缓存，再请求网络；GIF 数据会交给 ImageIO 拆帧并用 `UIImageView` 播放，搜索结果、图库封面和缩略图等预览场景固定渲染首帧静态图。翻到已缓存页时，阅读器会先用缓存索引直接恢复阅读页、图片 URL 和相邻页链接，不再请求阅读页 HTML；未命中缓存时才走网络解析。阅读目录同样优先使用对应页的缓存原图预览。

图片缓存由 `ImageCacheStore` 管理。缓存目录内保存图片二进制和 `index.json`，索引包含远端 URL alias、图库页码到图片缓存的映射，以及图库级标题、封面和总页数元数据。保存图片时会同时登记请求 URL 与最终响应 URL，避免重定向导致同图重复；刷新缓存时会按图片内容哈希清理旧重复文件、移除缺失文件对应的 alias 和页码记录，并把 alias 与页码记录重映射到保留的文件。缓存可全局清空，也可按图库删除对应页码记录和不再被引用的图片文件，设置页还可只清理搜索缩略图、封面等非图库阅读页图片。`GalleryDownloadManager` 负责整本后台下载：先按阅读页链接请求 HTML，再解析真实图片 URL 并写入带图库上下文的缓存；单页失败会记录错误并跳过该页继续下载后续页。下载进度读取时会用当前 `ImageCacheStore` 的真实图库页记录校准，避免缓存被清理后继续显示旧进度。

本地书架由 `LibraryStore` 通过 `UserDefaults` 保存历史、本地收藏和最近阅读页。`LibraryView` 将收藏拆成本地收藏与线上收藏：本地收藏读取 `LibraryStore`，线上收藏在 Cookie 已配置时复用 `SearchViewModel(initialSource: .favorites)` 请求站点 `favorites.php?favcat=all`，界面只保留关键词搜索。书架使用系统大标题，并让分段控件、线上收藏搜索框和翻页按钮作为滚动内容的吸顶分组 header，避免固定控件覆盖系统大标题；列表内容会先包进稳定容器再加外边距，避免 SwiftUI `Group` 把底部间距分发到每一行。阅读进度即使在用户没有先打开详情页时也会创建最小历史记录，避免书架页码落后于阅读器。本地书架只保存图库 URL、标题、缩略图 URL 和页码等轻量元数据，不保存远端 HTML、图片或用户凭据。

设置页通过共享的 `LibraryStore` 展示本地数据数量，并提供清空历史、收藏和阅读进度的确认操作。应用主题模式、主题颜色、阅读显示偏好和缩放倍率通过 `AppStorage` 在设置页、根视图和阅读器工具栏之间共享；主题默认跟随系统，也允许用户切换浅色或深色，主题色默认 `#00A8FF` 并可由用户通过系统颜色选择器自定义，根视图和设置页会同时把主题色应用到现代 `tint`、旧式 accent 控件和当前 UIKit window tint，设置页在主题色变化时会刷新列表以更新缓存的系统控件颜色。缓存策略说明放在本地数据区域上方。图片缓存页展示缓存图库数量、占用空间、按图库缓存管理入口、非图库缓存清理入口和确认清理入口；清理类确认弹窗挂在触发按钮自身，避免弹出位置远离用户刚点击的区域。缓存管理页列出已有缓存的图库，点击后进入对应图库页，也可滑动或通过菜单删除单个图库缓存。站点访问 Cookie 区域放在设置页底部，减少误触保存或清空凭据的机会。图库详情、缓存管理和标签搜索等二级页面会隐藏底部 Tab，避免和当前页面操作混在一起。

站点访问 Cookie 由 `SiteCookieStore` 保存到本机 Keychain。`URLSessionEHHTTPClient` 在请求公开站点页面时读取该 Cookie header 并注入请求，不在仓库内保存任何真实凭据。

## 约束

- App 不内置或提交站点上的内容资源；用户查看过的图片只作为本机运行时缓存保存，并可在设置页清理。
- `Info.plist` 允许 iPhone 横屏方向，阅读器额外提供手动横竖屏切换入口；自动旋转仍由系统方向感应处理。
- 用户凭据不写入仓库。
- 解析逻辑需要通过固定 HTML 样例测试，避免页面结构变动时静默失败。
- 真实页面结构变化时，先更新 `Docs/SITE_STRUCTURE.md`，再更新解析器和测试。
