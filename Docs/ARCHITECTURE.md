# 架构说明

## 分层

MyEHViewer 会按以下边界逐步建设：

- `App`：应用入口、根导航、全局主题偏好和通用文案。
- `Features/Search`：搜索输入、结果列表、高级搜索入口。
- `Features/Gallery`：图库详情、标签和元信息展示。
- `Features/Reader`：阅读器、图片页导航和阅读状态。
- `Networking`：HTTP 请求、Cookie header、请求头、响应错误和图片缓存。
- `Parsing`：HTML 到领域模型的解析。
- `Persistence`：历史、收藏、分享媒体和 Files 可见的永久图库文件。

## 当前状态

当前仓库包含工程骨架、领域模型、HTTP 客户端、基础解析层、搜索界面闭环、图库详情页、基础阅读器和本地书架。解析层已覆盖搜索列表、图库详情和阅读图片页的核心结构，并使用中性 HTML fixture 测试。

根导航由 `ContentView` 持有 `AppNavigationStore`，统一管理底部 Tab selection 和当前 `ReaderRoute`。阅读器不再是底部 Tab，而是从详情页、书架继续阅读和缓存管理等入口以全屏子页面呈现；关闭阅读器后会回到原来的上一级页面，避免底部 Tab 选中状态和当前阅读会话错位。

搜索界面由 `SearchViewModel` 负责浏览来源、筛选状态、请求、解析、分页、页码跳转、初始关键词搜索、初始浏览来源、最近搜索、失败重试 URL 和错误状态，`SearchView` 只负责 SwiftUI 展示与用户输入，并可在根页面和详情页导航栈复用。书架线上收藏复用同一个 `SearchViewModel(initialSource: .favorites)`，但只保留关键词搜索，并让分段控件和搜索框跟随列表正常滚动；线上收藏的上一页/下一页放在结果列表底部，和首页搜索结构保持一致。首页、热门、关注和站点收藏来源都通过 `EHSearchRequest` 生成 URL，页码跳转使用站点 zero-based `jump` 参数，分页按钮在窄屏下使用图标按钮和无障碍标签；搜索结果行会撑满列表宽度，避免不同标题长度导致缩略图和文字横向错位。

图库详情页由 `GalleryDetailViewModel` 负责详情请求、解析、缩略图分页合并、线上收藏状态读取和线上收藏表单提交，`GalleryDetailView` 展示封面、默认折叠的元信息、默认折叠且可继续搜索的标签、带缩略图的阅读页入口、后台下载入口和失败重试操作。进入图库页且详情尚未加载完成时，会显示不改变布局高度的轻量旋转加载提示，并尊重系统减少动态效果设置。收藏工具栏用菜单区分本地收藏和线上收藏；线上收藏会先请求 `gallerypopups.php` 弹窗，解析表单 action、隐藏字段、已选分类、移除选项和备注字段，再用 URL-encoded POST 提交，取消收藏时提交站点使用的移除分类值，Cookie 仍由共享 HTTP 客户端注入。详情页在已配置 Cookie 时会读取当前线上收藏状态，只有弹窗存在已选分类并带有移除分类、删除按钮或修改/移除收藏文案等已收藏标记时才视为已收藏，避免把站点默认选中的 `Favorites 0` 误判为全部图库已收藏；已收藏图库会在标题区域显示线上收藏徽标和分类名。图库页面预览会优先使用对应页的本地缓存原图；未缓存时才使用站点缩略图和 CSS sprite 裁剪数据。详情页远端加载失败时，如果缓存索引中仍有该图库的页码记录，会直接展示本地已缓存页面入口，让 404 或离线场景仍可进入阅读器查看已有图片。

阅读器由 `ReaderViewModel` 负责图片页请求、解析、翻页、已知页码范围、页面目录自动补齐、页码入口跳转状态和图片资源重载 token，`ReaderView` 展示当前图片、返回上一级、页码、上一页、下一页、缩略图目录、页码输入跳转、图片加载失败重试、缩放控制和显示偏好，顶部工具栏不再暴露外部浏览器入口。阅读器默认隐藏主要控制区，只显示图片和一个左上角返回目标；用户点击左侧/右侧翻页，点击中间显示控制 UI，并可通过双指缩放临时放大。长按当前图片会先弹出确认操作，再通过 Photos add-only 权限把当前图片原始数据保存到系统相册，保存时优先读取本地文件。上一页/下一页会按当前页码优先查已知图库目录和本地索引，再回退到阅读页 HTML 解析出的相邻链接，因此只有第一页/最后一页才应禁用相应按钮。远端图片通过 `CachedRemoteImageView` 统一加载，依次查找 `CachedGalleryStore` 的永久图库文件、`ImageCacheStore` 的临时磁盘缓存，最后才请求网络；GIF 数据会交给 ImageIO 拆帧并用 `UIImageView` 播放，搜索结果、图库封面和缩略图等预览场景固定渲染首帧静态图，并按目标像素尺寸降采样，避免把本地原图完整解码成小预览。翻到本地页时，阅读器会先用合并后的图库索引直接恢复阅读页、图片 URL 和相邻页链接，不再请求阅读页 HTML；未命中本地文件时才走网络解析。阅读目录同样优先使用对应页的本地原图预览。

图片缓存由 `ImageCacheStore` 管理。缓存目录内保存图片二进制和 `index.json`，索引包含远端 URL alias、图库页码到图片缓存的映射，以及图库级标题、封面、总页数和 404 下载标记。保存图片时会同时登记请求 URL 与最终响应 URL，避免重定向导致同图重复；保存路径只做轻量刷新，不再每写入一张图就全量读取并哈希整个缓存目录。同步和异步图片写入、文件删除、index 编码与原子替换共用同一个 FIFO utility 写入器，下载路径在调用点立即入队并异步等待，用户主动清理仍保持同步可见。异步保存同时记录全局/图库清理 generation、每个 cache key 的调用序号和成功序号，已经跨过 `await` 的旧保存不能在清理或更新写入后恢复旧索引；如果失效事务已经替换物理 bytes，则按磁盘最新版本更新 digest、大小、所有共享页面记录、统计和 index snapshot，或在没有 alias、页面和有效 pending owner 时删除孤儿文件。完成一个永久图库时按已知 cache key 增量更新内存统计，不再在 `MainActor` 重扫缓存和永久目录；下载批次进行时，设置、缓存和统计页的周期性磁盘刷新会跳过，避免页面切换触发全盘目录扫描。缓存读取会先解析本地文件 URL，再在后台读取并解码成 `UIImage` 后交给界面展示，减少列表滚动和阅读翻页时的大块 `Data` 持有、主线程 IO 与解码压力。缓存可全局清空，也可按图库删除对应页码记录和不再被引用的图片文件，设置页还可只清理搜索缩略图、封面等非图库阅读页图片。

`GalleryDownloadManager` 负责整本后台下载：先按阅读页链接请求 HTML，再解析真实图片 URL，并用阅读页 URL 作为图片请求 `Referer` 写入带图库上下文的永久暂存区。队列保留最多 5 个逻辑图库任务和单图库最多 3 个页面任务，但所有图库共享默认容量 4 的完整页面流水线许可，限制同时持有的大图数据、网络解析和持久化压力；图片页任务按完成顺序补位，避免较慢页面拖住后续页面。页面进度由任务内的唯一成功页码集合累计，不依赖节流后的图库摘要，因此小图库快速完成也能正确 finalize。网络和 5xx/429 等临时失败可重试，404、解析失败、取消和持久化失败不会重新下载已经成功获取的图片。完整暂存可在批量恢复时直接离线 finalize，不请求图库 HTML 或图片，即使清单保留历史 404 标记也不影响完成；只有已经 finalize 且永久页码完整的图库会阻止重复下载，完整临时缓存仍可继续提升为永久图库。暂停会取消许可等待者，并为每次图库运行分配独立执行标识，旧任务晚退出时不能清理或覆盖刚恢复的新任务。缓存管理页可把所有仍需下载或 finalize 的图库加入下载队列，并在有任务进行时显示总体页数进度、活动/排队任务数和平均下载速度；下载进度观察被限制在详情页的小型状态控件和缓存管理下载面板，不会让页面目录或缓存网格随每个进度 tick 整体刷新。

`CachedGalleryStore` 管理 `Documents/Cached Gallery` 下一个图库一个文件夹的永久资源，并在 `Application Support/CachedGalleryStaging` 保存可恢复的中间状态。每个图库目录包含按页码命名的原格式图片和 `manifest.json`。文件内的 `PersistenceCoordinator` actor 串行拥有最新 staging manifest，负责准备目录、批量导入、图片格式识别、页面/manifest 原子写入、磁盘刷新和 finalize；同图库并发页面不会用旧 manifest 覆盖新页面、备注或 404 状态。`CachedGalleryStore` 在 `MainActor` 只维护阅读器同步查询需要的轻量投影，逐页事务只替换目标页和标记目标图库摘要为 dirty，准备、导入和 finalize 才重建单个图库，启动恢复和显式刷新时才扫描完整目录。刷新和异步事务都带图库 generation/revision，删除或更新产生的 tombstone 会阻止更旧结果恢复已移除状态。暂存完整性严格要求页码覆盖 `1...totalPageCount`；完整暂存可幂等 finalize，即使另一个执行已经移动目录也会验证正式目录并返回一致投影。已有缓存按图库批量导入并只写一次 manifest；迁移成功后 `ImageCacheStore` 才删除不再需要的缓存副本，失败或中断时原缓存保持不变。明确触发的整本下载直接写入永久暂存区，普通浏览产生的图片仍使用可清理缓存。

`HitomiDataSource` 对同一图库的 galleryinfo 请求进行在途合并，并保留容量 8 的内存 LRU。每个调用方拥有独立 waiter：单个调用取消只结束自己的等待，最后一个 waiter 离开时才取消底层请求；批量加载会继续跳过普通失败，但会立即传播调用方取消。请求成功后按图库发布并复用解析结果，失败不会进入缓存。

本地书架由 `LibraryStore` 通过 `UserDefaults` 保存历史、本地收藏和最近阅读页。`LibraryView` 将收藏拆成本地收藏与线上收藏：本地收藏读取 `LibraryStore`，线上收藏在 Cookie 已配置时复用 `SearchViewModel(initialSource: .favorites)` 请求站点 `favorites.php?favcat=all`，界面只保留关键词搜索。书架使用系统大标题，分段控件和线上收藏搜索框跟随列表正常滚动，线上收藏翻页按钮放在结果列表底部；列表内容会先包进稳定容器再加外边距，避免 SwiftUI `Group` 把底部间距分发到每一行。阅读进度即使在用户没有先打开详情页时也会创建最小历史记录，避免书架页码落后于阅读器。本地书架只保存图库 URL、标题、缩略图 URL 和页码等轻量元数据，不保存远端 HTML、图片或用户凭据。

设置页通过共享的 `LibraryStore` 展示本地数据数量，并提供清空历史、收藏和阅读进度的确认操作。应用主题模式、主题颜色、阅读显示偏好和缩放倍率通过 `AppStorage` 在设置页、根视图和阅读器工具栏之间共享；主题默认跟随系统，也允许用户切换浅色或深色，主题色默认 `#00A8FF` 并可由用户通过系统颜色选择器自定义，根视图和设置页会同时把主题色应用到现代 `tint`、旧式 accent 控件和当前 UIKit window tint，设置页在主题色变化时会刷新列表以更新缓存的系统控件颜色。缓存策略说明放在本地数据区域上方，图片缓存区域放在本地数据区域之前。图片缓存页展示缓存图库数量、占用空间、按图库缓存管理入口、非图库缓存清理入口和确认清理入口；清理类确认弹窗挂在触发按钮自身，避免弹出位置远离用户刚点击的区域。缓存管理页列出已有缓存的图库，顶部提供继续未完成下载按钮和进行中下载进度区，点击图库后优先进入本地缓存入口页，可直接打开缓存阅读器，也可再进入远端图库详情页；列表支持滑动或通过菜单删除单个图库缓存。站点访问 Cookie 区域放在设置页底部，减少误触保存或清空凭据的机会。图库详情、缓存管理和标签搜索等二级页面会隐藏底部 Tab，避免和当前页面操作混在一起。

站点访问 Cookie 由 `SiteCookieStore` 保存到本机 Keychain。`URLSessionEHHTTPClient` 在请求公开站点页面时读取该 Cookie header 并注入请求，不在仓库内保存任何真实凭据。

## 约束

- App 不内置或提交站点上的内容资源；普通浏览图片只作为可清理缓存保存，用户明确下载或迁移的图库永久保存在本机 `Cached Gallery` 文件夹。
- `Info.plist` 允许 iPhone 横屏方向，阅读器额外提供手动横竖屏切换入口；自动旋转仍由系统方向感应处理；相册保存仅申请 add-only 写入用途说明。
- 用户凭据不写入仓库。
- 解析逻辑需要通过固定 HTML 样例测试，避免页面结构变动时静默失败。
- 真实页面结构变化时，先更新 `Docs/SITE_STRUCTURE.md`，再更新解析器和测试。
